# Employee Management System Documentation

## Project Overview

The Employee Management System is a Java console-based application used to manage employee records efficiently.

The system demonstrates the use of:

- Java Collections (ArrayList and HashMap)
- File Handling and Serialization
- CRUD Operations
- Sorting and Searching
- Input Validation
- Report Generation

The application allows users to add, view, search, update, delete, and sort employee records and generate statistical reports.

---

## Setup Instructions

1. Install Java JDK.
2. Open the project folder in VS Code.
3. Compile the project using:
   javac -d . src/com/*.java
4. Run the program:
   java com.EmployeeManagementSystem

### Folder Description

**src/com**  
Contains all the Java source code files of the Employee Management System.

- `Employee.java` – Defines the Employee class and its attributes.
- `EmployeeFileHandler.java` – Handles file saving and loading operations.
- `EmployeeManagementSystem.java` – Main program that manages menu operations and employee actions.
- `EmployeeReportGenerator.java` – Generates salary and department reports.

**data**  
Stores the serialized employee data file (`employees.dat`) used for persistence.

**docs**  
Contains project documentation and visual explanations of the system.

- `documentation.md` – Technical documentation of the project.
- `visual_documentation.md` – Screenshots showing system functionality.
- `screenshots/` – Images of the application outputs.

**reports**  
Contains example output files generated by the system.

**README.md**  
Provides an overview of the project and instructions for running the application.

## Data Format

Each employee object stores the following fields:

| Field      | Type    |
|------------|---------|
| id         | String  |
| name       | String  |
| department | String  |
| position   | String  |
| salary     | double  |
| joinDate   | Date    |

Employee data is stored using **Java serialization** in:
data/employees.dat


---

## File Handling

The system uses:

- `ObjectOutputStream` → save employee data
- `ObjectInputStream` → load employee data

Data is automatically saved when employees are added, updated, or deleted.

---

## Employee Management Workflow

1. Program starts
2. Employee data loads from file
3. User selects an operation from the menu
4. System performs the requested action
5. Data is saved automatically

Operations available:

- Add Employee
- View Employees
- Search by ID / Name / Department
- Update Employee
- Delete Employee
- Sort Employees
- Generate Reports

---

## Example Code Snippet

Example of sorting employees by salary:

employees.sort((a,b) -> Double.compare(a.getSalary(), b.getSalary()));

This sorts the employee list in ascending order of salary.
